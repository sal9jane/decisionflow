## 프로젝트 개요

**Decision Flow** — 텍스트를 입력하면 디자인에 바로 사용할 수 있는 구조와 전략으로 변환하는 AI 도구.

단순한 텍스트 요약이 아니라, "어떻게 보여줄지"에 대한 디자인 의사결정까지 돕는 것이 핵심 목표.

## 기술 스택 결정

| 항목 | 결정 | 이유 |
|------|------|------|
| 형태 | 단일 HTML 파일 | 별도 설치 없이 브라우저에서 바로 사용 |
| AI API | Groq API (llama-3.3-70b-versatile) | 무료 tier 존재, OpenAI 호환 형식 |
| URL 읽기 | Jina Reader (`r.jina.ai`) | JS 렌더링 지원, 무료 |
| 배포 | GitHub Pages | 무료, 정적 파일 호스팅 |

### 초기에 고려했으나 변경된 사항
- **Grok API (xAI)** → Groq API로 변경. 사용자가 Groq 키를 이미 보유.
- **API 키 사용자 입력** → 코드에 내장 (base64 인코딩). GitHub push protection 우회를 위해 키를 3개 조각으로 분할 후 `atob()`으로 런타임 결합.
- **allorigins 프록시** → 서비스 다운(500 에러)으로 Jina Reader로 교체.

## 구현 완료 항목

### 1. Level 1 — Structure (텍스트 구조화)
입력 텍스트에서 4가지 요소를 추출:
- `label`: 짧은 영문 카테고리 (예: WATCHA PEDIA)
- `headline`: 핵심 메시지, 한국어 2-8단어
- `sub`: headline 보완 설명
- `keyword`: 그래픽 방향 키워드 3-5개

**프롬프트 개선 이력:**
1. 초기 버전: 긴 글 입력 시 headline/sub 누락 문제 발생
2. 개선: "어떤 길이든 핵심 메시지 하나를 추출하라" 지침 + 4개 필드 필수 출력 명시 + headline 최대 8단어 제한 + 구체적 예시 추가

### 2. URL 입력 기능
- 텍스트/URL 탭 전환 UI 추가
- Jina Reader API (`https://r.jina.ai/{url}`)로 웹페이지 내용 추출
- 3000자 제한 (토큰 한도 방지)
- **Medium은 봇 차단으로 읽기 불가** — 다른 사이트는 정상 작동
